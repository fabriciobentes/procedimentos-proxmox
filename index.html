Nginx

Servidor web e proxy reverso de alta performance.

Instalar

apt
apt update && apt install -y nginx
systemctl enable nginx
systemctl start nginx
verificar
nginx -v
curl -I http://localhost

Proxy reverso — template

/etc/nginx/sites-available/app
server {
    listen 80;
    server_name app.exemplo.local;

    location / {
        proxy_pass         http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header   Upgrade $http_upgrade;
        proxy_set_header   Connection 'upgrade';
        proxy_set_header   Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
ativar site
ln -s /etc/nginx/sites-available/app /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx

Diagnóstico

status
systemctl status nginx
nginx -t
tail -n 50 /var/log/nginx/error.log

Backup & Restore

Transferência, união, restauração, importação e geração de backups no Proxmox.

Enviar backup do PC para o Proxmox

Todo backup vindo do Windows deve ser enviado primeiro para o host Proxmox, normalmente para o diretório /root.

destino padrão
/root

Enviar via WinSCP

conexão
Host: 10.170.7.x
Port: 22
User: root
Password: senha do host
Padrão: Após conectar no WinSCP, envie o arquivo de backup para /root.

Enviar via SCP (PowerShell / CMD)

scp
scp "C:\caminho\do\backup.vma.zst" root@10.170.7.x:/root/

Caminho com espaço no Windows

exemplo
scp "C:\Users\Fabricio\Desktop\Backup VM\backup.vma.zst" root@10.170.7.x:/root/

Unir backup fragmentado

Use este processo quando o backup estiver dividido em múltiplas partes (.001, .002, .003...).

unir partes
cat backup.vma.zst.0* > backup-final.vma.zst
validar
ls -lh

Validar envio

verificar
ls -lh /root

Restore de backup Proxmox (.vma / .vma.zst)

Backups exportados do Proxmox normalmente usam formato .vma ou .vma.zst.

qmrestore
qmrestore /root/backup.vma.zst 103

Restore com storage definido

storage
qmrestore /root/backup.vma.zst 103 --storage local-lvm

Restore sobrescrevendo VM existente

force
qmrestore /root/backup.vma.zst 103 --storage local-lvm --force

Descompactar backup .zst

zstd
zstd -d /root/backup.vma.zst

Gerar backup manual

vzdump
vzdump 103 --storage local --compress zstd

Importar disco manual (.qcow2 / .img)

Use este fluxo quando o backup não for um backup nativo do Proxmox, mas apenas um disco exportado.

enviar disco
scp "C:\caminho\disco.qcow2" root@10.170.7.x:/root/

Criar VM vazia

qm create
qm create 103 --name ubuntu --memory 4096 --cores 2 --net0 virtio,bridge=vmbr0

Importar disco

importdisk
qm importdisk 103 /root/disco.qcow2 local-lvm

Anexar disco importado

anexar
qm set 103 --scsihw virtio-scsi-pci --scsi0 local-lvm:vm-103-disk-0

Definir boot

boot
qm set 103 --boot order=scsi0

Recovery

Recuperação, diagnóstico e reparo de falhas críticas no host Proxmox.

FSCK offline (modo manutenção)

Use este processo quando o sistema não inicializar corretamente, entrar em emergency mode ou apresentar corrupção no sistema de arquivos.

listar discos
lsblk
desmontar
umount /dev/sdXn
reparar
fsck -f -y /dev/sdXn

Forçar verificação no boot

touch
touch /forcefsck
reboot
Atenção: Nunca rode fsck em partição montada.

Verificar pools ZFS

zpool
zpool status
zpool import

Importar pool manualmente

import
zpool import -f rpool

Importar pool em modo alternativo

altroot
zpool import -f -R /mnt rpool

Scrub e integridade

scrub
zpool scrub rpool
zpool status

Diagnóstico avançado

zdb
zdb -l /dev/sdX
zdb -C rpool

Boot em Rescue Mode

Use Rescue Mode quando o Proxmox não iniciar normalmente e for necessário recuperar o sistema manualmente.

rescue
Boot pelo ISO do Proxmox
Advanced Options
Rescue Boot

Montar sistema manualmente

mount
mount /dev/sdXn /mnt
mount --bind /dev /mnt/dev
mount --bind /proc /mnt/proc
mount --bind /sys /mnt/sys
chroot /mnt
